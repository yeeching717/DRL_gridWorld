# Grid World Lab

Demo: https://ca3bffa1-9a53-4daf-bfc2-a2603efe3aaf-00-380cywq22a85i.pike.replit.dev/

這是一個可互動的 Grid World 視覺化小專案，使用原生 HTML、CSS、JavaScript 實作，功能包含地圖編輯、隨機策略評估與價值迭代求最佳策略。

## 功能總覽

1. 網格地圖開發
- 支援設定網格大小 n，範圍 5 到 9。
- 可點擊設定起點（綠色）與終點（紅色）。
- 可放置障礙物（灰色），數量可為 0 到 n-2。

2. 策略顯示與價值評估
- 可為每個可行狀態隨機產生策略行動（上下左右箭頭）。
- 使用策略評估計算每個狀態的價值函數 V(s)。

3. 價值迭代與最佳政策
- 實作價值迭代（Value Iteration）。
- 迭代後顯示每格最佳行動（覆蓋原本隨機策略）。
- 更新並顯示最佳策略下的 V(s)。
- 額外顯示從起點推導出的最佳路徑（藍色格子）。

4. 矩陣輸出
- 同時顯示 Policy Matrix。
- 同時顯示 Value Matrix。

## 專案結構

- index.html：頁面結構與控制面板。
- style.css：介面樣式與格子視覺設計。
- script.js：Grid World 邏輯、策略評估、價值迭代、最佳路徑推導。

## 操作流程

1. 設定 n 並建立地圖。
2. 使用模式切換設定起點、終點、障礙物。
3. 點選「隨機策略 + 策略評估」檢視隨機政策與 V(s)。
4. 點選「價值迭代找最佳策略」檢視最佳政策、更新後 V(s) 與藍色最佳路徑。

## 演算法設定（可在 script.js 調整）

- 折扣因子 gamma：0.9
- 一般步驟回饋 step reward：-1
- 抵達終點回饋 goal reward：20
- 收斂門檻 theta：1e-4

## 常見問題

1. 為什麼按演算法按鈕沒有更新？
- 請先設定起點與終點。
- 障礙物數量不可超過 n-2。

2. 為什麼最佳路徑沒有一路走到終點？
- 若策略遇到循環（例如等價行動造成來回），路徑會在偵測到重複狀態後停止，這是正常保護機制。
